# Структура проекта

Проект приведён к структуре, похожей на пример из папки `src`:

```text
src/
├── app/
│   ├── globals.css
│   ├── layout.tsx
│   └── page.tsx
├── components/
│   ├── Header/
│   │   └── Header.tsx
│   └── ui/
│       └── index.tsx
├── data/
│   ├── buildingPassportData.ts
│   └── types.ts
├── pages/
│   └── DashboardPage/
│       ├── DashboardPage.tsx
│       ├── components/
│       │   ├── ProgressCard.tsx
│       │   ├── ProjectOverview.tsx
│       │   └── StatusEditor.tsx
│       └── sections/
│           ├── DocumentsSection.tsx
│           └── PhotoReport.tsx
└── utils/
```

## Логика разделения

- `app` — файлы Next.js: layout, глобальные стили и входная страница.
- `components` — общие компоненты, которые можно переиспользовать в разных страницах.
- `data` — моковые данные и типы проекта.
- `pages/DashboardPage` — основная страница цифрового строительного паспорта.
- `components` внутри страницы — маленькие блоки страницы.
- `sections` внутри страницы — крупные секции страницы.

## Адаптивность

Дополнительно улучшена адаптивность интерфейса:

- уменьшены отступы карточек и страницы на маленьких экранах;
- карточка объекта перестраивает статистику из одной колонки в три по мере роста ширины;
- блок прогресса больше не сжимает круговую диаграмму и текст на телефонах;
- документация отображается карточками на мобильных и таблицей на больших экранах;
- фотоотчёт плавно меняет сетку: 1 / 2 / 3 колонки;
- шапка защищена от переполнения длинными статусами;
- выпадающий список статуса ограничен шириной экрана и не вылезает за края viewport.
